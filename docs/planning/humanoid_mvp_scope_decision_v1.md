# 6/10 MVP スコープ決定書 v1

> 本書はMVPスコープの**唯一の確定版**。他のドキュメントと矛盾があれば本書を優先する。

---

## 1. 目的

6/10のロボスタで名刺代わりに見せる。
**網羅性ではなく、質と本気度を伝えることが唯一の目標。**

---

## 2. 作るもの（確定）

| ページ | URL | 優先度 |
|---|---|---|
| Home | `/` | 必須 |
| About | `/about` | 必須 |
| 旗艦ガイド | `/guides/decision-variables` | 必須・最優先 |
| Contact | `/contact` | 必須 |
| データモデル定義 | `robots / guides / reports / use-cases / manufacturers` | 必須（安い・先行） |
| ロボットレコード1件 | `/robots/[slug]` | 余力があれば |

---

## 3. 作らないもの（確定）

以下は**6/10には作らない**。データモデルだけ先行し、中身は後。

- robots一覧ページ・フィルタUI
- 企業/代理店DB
- 比較表
- 用途から探す（use-cases）
- 事例DB（cases）
- reports連投
- services / works
- DB、ログイン、管理画面

---

## 4. 成功条件

- [ ] 30秒で「何のサイトか・誰向けか」が伝わる
- [ ] 旗艦記事を読んで「買い手目線の深さ」が伝わる
- [ ] Aboutで「運営者の本気度」が伝わる
- [ ] Contactから連絡できる
- [ ] 空の棚がない（ダミー文・未完成ページはナビから外す）

---

## 5. 技術判断（確定）

| 項目 | 選択 |
|---|---|
| フレームワーク | Next.js |
| コンテンツ | 当面はローカルデータ。将来CMS接続前提 |
| CSS | Tailwind CSS可。ただしトークン設計で制御 |
| フォーム | メール直リンク または Formspree |
| DB | なし |
| CMS | 今はなし。Sanity / microCMS を有力候補として見据える |
| 公開 | Vercel |

---

## 6. グローバルナビ（MVP版）

出すのは中身がある物だけ。

```
導入ガイド / このサイトについて / 相談
```

ロボットレコードが1件でも完成したら `ロボット` を追加してよい。なければ出さない。

---

## 7. エンプティステートの扱い

「近日公開」ページ等で予告している箇所には、必ず以下のいずれかを仕込む。

- 更新通知の登録フォーム（主CTA）
- 掲載・取材募集の一文 ＋ Contact導線
- SNSフォロー誘導

「まだありません」のみの表示は出さない。

---

## 8. 6/10後に追加するもの（優先順）

1. robots一覧 + 詳細ページ（5〜6機種・日本向けフィールドまで深く）
2. メーカー/供給体制DB（代理店・SIerを含む）
3. 比較表（製品DBから生成）
4. 用途から探す
5. 事例DB（海外事例の集約→国内一次取材へ）

---

## 9. 判断基準

迷ったらこの一文で判断する：

**「6/10の名刺として、プロに質と本気度が伝わるか？」**

Yesなら進める。Noなら削る。判断できなければ削る。

# 画像フォールバック・URLエラー修正 実装計画

作成日: 2026-06-02

## 1. 目的

現在、ロボットおよびメーカーの画像（ロゴ・ヒーロー画像）において以下の問題が発生している。これを解消し、信頼性の高いB2BサイトとしてのUI品質を確保する。

1. **無効なURL (404/403)**: 一部の直リンク画像（Teslaのロゴ等）がリンク切れ、またはアクセス拒否により非表示となっている。
2. **プレースホルダーの品質**: 画像がない場合のフォールバック表示が単なるテキスト等になっており、「公式画像を申請中・準備中である」というプロフェッショナルなニュアンスが伝わらない。
3. **Next.js の画像最適化設定**: `next/image` コンポーネントを使用する場合に必要な外部ドメインの許可リスト（`remotePatterns`）が設定されていない。

## 2. 既存コードとの整合性・原則確認

- **DRY / KISS**: 画像フォールバックのUI（プレースホルダー）は共通のアイコンやスタイルを使用し、個別のコンポーネントで複雑な条件分岐を重複させない。
- **Single Source of Truth**: フォールバック用のテキスト（「公式画像申請中」等）は `lib/uiText.ts` に集約する。
- **Operational Safety**: 404/403エラーとなるURLは無理に残さず、データ側（`data/*.ts`）でクリーンに削除（`undefined`扱い）し、安全なプレースホルダー表示に倒す。
- **URL State / SEO**: `<img>` タグの `alt` 属性は適切に保ち、アクセシビリティを損なわないようにする。

## 3. 調査済みファイルと状態

- `data/manufacturers.ts`, `data/robots.ts`: URLの直書きが存在。一部無効。
- `lib/media.ts`: `canDisplayAsset` と `getDisplayableAsset` により、ポリシーベースで画像オブジェクトをフィルタリングする処理は健全に動作している。
- `components/RobotCard.tsx`: `<img src={...}>` を使用。フォールバックは `uiText.robots.mainImageMissing` のテキストのみ。
- `components/ManufacturerLogoName.tsx`: `<img src={...}>` を使用。フォールバックは頭文字2文字。
- `lib/uiText.ts`: プレースホルダー文言の定義場所。
- `next.config.mjs`: `images.remotePatterns` の設定なし。現状は標準の `<img>` を使っているためエラーにはなっていないが、将来の最適化を見据えて設定するか、標準 `<img>` で統一するかの判断が必要。→ **KISS原則に基づき、今回は現状の標準 `<img>` タグの利用を維持し、Next.jsの設定追加は見送る（データとUIの修正に集中する）。**

## 4. 実装手順（AI実行用）

### Step 1: データクレンジング (`data/manufacturers.ts`, `data/robots.ts`)
- 事前の `curl` 調査で 404 または 403 エラーが確定している画像URLのオブジェクト（`logo` または `heroImage`）を削除（コメントアウトまたはプロパティ自体を削除）する。
  - 対象例: `tesla` のロゴ（Wikimedia URLが404）、`kepler-robotics` のロゴ（ドメイン解決不可）、`figure-ai` のロゴ（Contentful 404）など、無効と判明したもの。
  - 無理に代替URLを探さず、まずは「画像なし（プレースホルダー）」の状態を正しく作れるようにする。

### Step 2: UIテキストの更新 (`lib/uiText.ts`)
- 画像が存在しない場合の文言を、より誠実でプロフェッショナルなものに変更する。
- 修正対象: `robots.mainImageMissing` などを `"No Image"` や `"Official Image Requested" (公式画像申請中)` に更新。

### Step 3: フォールバックUIの改修 (`components/RobotCard.tsx`)
- `getDisplayableAsset` が `undefined` を返した場合のUIをリッチにする。
- 単なるテキストではなく、薄いグレーの背景（`bg-neutral-100`等）の中央にカメラアイコン（`lucide-react`の `CameraOff` 等）と、「公式画像申請中」のテキストを配置する。

### Step 4: （任意）詳細ページ等、他の画像表示コンポーネントの確認
- `RobotCard.tsx` 以外で画像を使用している箇所（例: 詳細ページのカルーセルなど）があれば、同様にフォールバックが機能するか確認する。

## 5. 影響範囲とリスク
- **影響範囲**: サイト全体のロボットカード、メーカー名表示部分。
- **リスク**: 特になし。コンポーネントの描画ロジックの軽微な変更に留まるため、既存の機能や検索処理には影響を与えない。

## 6. 検証方法

- **コマンド検証**:
  ```bash
  npm run validate:data
  npm run build
  ```
- **手動（自己）確認項目**:
  - 画像URLを削除したロボット/メーカーのカードで、意図したプレースホルダー（アイコン＋文言）が崩れずに表示されているか。
  - 画像URLが有効なものは、従来通り画像が表示されているか。
  - ビルド時に不要な lint エラーや型エラーが発生していないか。

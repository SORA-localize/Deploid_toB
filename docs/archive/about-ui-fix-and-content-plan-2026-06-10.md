# /about UI修正 + 不足コンテンツ追記 実装計画

作成日: 2026-06-10  
監査反映: 2026-06-10

---

## 目的

1. `/about` ページの UI を確立済みデザイン基準に合わせ、"AI 感" を排除する
2. 運営者情報・出典ポリシー・掲載方針など、信頼性に必要なコンテンツを既存ページへ追記する
3. ContactForm でメールを収集しているため、プライバシーポリシーページを新設する

## UI調査レポート（/about/page.tsx）

### 問題一覧

| # | カテゴリ | 問題 | 箇所 | 重要度 |
|---|---------|------|------|--------|
| 1 | タイポ | h1 にレスポンシブなし（`text-3xl` 固定） | `page.tsx:43` | 中 |
| 2 | タイポ | ロードマップ h2 が `text-2xl`（基準は `text-xl`） | `page.tsx:64` | 中 |
| 3 | デザイン | 3-card 等幅の value カード群 — 「アイコン付き角丸カード3枚の特徴セクション」の構造的同型。アイコン・角丸はないが gestalt が AI 感 | `page.tsx:51-61` | 高 |
| 4 | レイアウト | ページ外コンテナが `py-6 sm:py-10` — 他ページ基準は `py-8` | `page.tsx:38` | 低 |
| 5 | 構造 | CTA 末尾セクションが `bg-card p-8` で二重ネスト（セクション内にカード）— 他セクションと構造が揃っていない | `page.tsx:81-94` | 中 |
| 6 | 内容 | 運営者情報（氏名 / 連絡先）がゼロ | — | 高 |
| 7 | 内容 | 出典ポリシー（reliability 基準の説明）がない | — | 高 |
| 8 | 内容 | 掲載・画像方針が footer notice の 1 文だけ | — | 中 |
| 9 | 内容 | ContactForm でメール収集しているがプライバシーポリシーへのリンクがない | `ContactForm.tsx` | 高（法的） |

### 詳細メモ

- **#1**: 確立基準は `text-2xl md:text-3xl font-semibold leading-tight`。現状は `text-3xl` 固定で小画面でも大きく表示される。
- **#2**: ページ最上位節の h2 基準は `text-xl font-semibold`（ManufacturerDetailSection 等）。`text-2xl` は過剰。
- **#3**: 3枚等幅カードはどんな内容でも「サービス紹介テンプレ」に見える。ミッション/課題/方針は階層があるため、`<dl>` ベースの定義リスト形式にすると情報の重みが正しく伝わる。
- **#5**: 末尾 CTA は他セクションと同様に `py-8 border-b border-border`（または `py-8`）のフラット構造にする。内側カードは不要。

### 参照した確立済みパターン

- `text-2xl md:text-3xl font-semibold leading-tight` — 詳細ページ h1
- `text-xl font-semibold` — 最上位節 h2（ManufacturerDetailSection 他）
- `py-8 border-b border-border` — 標準セクション縦余白
- `site-container py-8` — ページ外コンテナ

## フェーズ構成

### Phase 1 — /about UI修正

**変更ファイル**: `src/app/about/page.tsx` のみ  
**変更しないファイル**: globals.css / lib / components（依存なし）  
**影響範囲**: コード上の変更範囲は `/about` に限定。CTA からの `/contact` への導線やページ全体の見え方は変わる

#### 変更内容

1. **h1 レスポンシブ追加**  
   `text-3xl` → `text-2xl md:text-3xl`

2. **ロードマップ h2 修正**  
   `text-2xl font-semibold` → `text-xl font-semibold`

3. **values セクション再構成**  
   3 カードグリッドを廃止し、`<dl>` ベースの左揃え定義リスト形式へ。  
   - 左カラム: `text-xs font-medium uppercase tracking-wide text-muted-foreground` のラベル  
   - 右カラム: 見出し（`font-semibold text-foreground`）+ 本文  
   - レイアウト: `grid grid-cols-1 md:grid-cols-[8rem_1fr]` × 3行  
   → 「強いタイポ階層・左揃え・グリッド規律」を体現、AI 感を排除

4. **CTA セクション簡素化**  
   `bg-card p-8` ラッパーを外し、他セクションと同様の `py-8` フラット構造へ。  
   見出し + テキスト + ボタンをそのまま配置。

5. **コンテナ py 統一**  
   `py-6 sm:py-10` → `py-8`

---

### Phase 2 — /about コンテンツ追記

**変更ファイル**: `src/app/about/page.tsx`  
**前提**: 運営者情報（名前・メールアドレス・公開開始時期）をユーザーから確認してから実装

#### 追加セクション一覧

| セクション | 内容 | 挿入位置 |
|---|---|---|
| **運営者情報** | 運営者名・問い合わせメール・公開開始時期 | ロードマップ の下 |
| **出典・信頼度ポリシー** | `official / press / estimated / 要確認` の定義と使用基準をユーザー向けに説明 | 運営者情報 の下 |
| **掲載・画像方針** | 掲載基準・中立性の担保・メーカー画像著作権の扱い（footer notice を展開した版） | 出典ポリシー の下 |

#### Phase 2 実装前に確認が必要な情報

- [ ] 運営者名として出す表記（個人名 / 屋号 / 法人名）
- [ ] 公開するメールアドレス（例: contact@deploid.jp など）
- [ ] サイト公開開始の時期（年月）

### Phase 3 — プライバシーポリシーページ新設

**新規ファイル**: `src/app/privacy/page.tsx`

**変更ファイル**:
- `lib/siteNavigation.ts` — `siteNavItems` には追加しない。`footerNavItems` の生成ロジックを変更し、`/privacy` をフッター専用で追加する（`Header` は `siteNavItems` をそのまま使うため、`siteNavItems` に入れるとヘッダーにも表示される。`Header.tsx:65`, `siteNavigation.ts:7` 参照）
- `components/ContactForm.tsx` — フォーム送信ボタン下に「プライバシーポリシーに同意の上、送信してください」リンク付き 1 行を追加

**内容骨子**:
- 収集する情報: 氏名（任意）・会社名（必須）・メールアドレス（必須）・問い合わせ種別・問い合わせ内容（必須）
- 利用目的: 問い合わせへの対応のみ
- 第三者提供: しない（Formspree 経由の送信であることを明示）
- 保管期間: 問い合わせ対応完了後、速やかに削除
- 開示・訂正・削除の問い合わせ先: 運営者メールアドレス

**備考**: Formspree の Privacy Policy URL は実装直前に公式サイトで現行 URL を確認してから記載する（外部 URL は変更される可能性がある）

```
Phase 1 → npm run build → 手動確認 → commit
Phase 2 → 問い合わせメールアドレス確定後 → npm run build → 手動確認 → commit
Phase 3 → 問い合わせメールアドレス確定後（Phase 2 全完了は不要）→ npm run build → 手動確認 → commit
```

Phase 2・3 の共通ブロッカーは「公開する問い合わせメールアドレスの確定」のみ。運営者名・公開時期は Phase 2 のみに影響し、Phase 3 は独立して進められる。
